## 4) Findings (uu tien theo muc do)

### [High] CLI that khong chay duoc voi config hien co
- Trieu chung:
  - Chay `node src/cli.js web` bi dung do validate yeu cau `config.paths`.
- Nguyen nhan kha nang cao:
  - Config schema trong `sao.config.json` dang theo dang cu (`root`, `temp`) trong khi code validate dang yeu cau schema moi (`paths`, `compiled`).
- Tac dong:
  - Block luong compile qua CLI trong workspace hien tai.
  - Tao do lech giua "test pass" va "van hanh that fail".

### [High] Tat ca lenh --version cua CLI deu bi vo
- Trieu chung:
  - `src/cli.js`, `src/build-cli.js`, `src/dev-cli.js` deu `require('./package.json')` tu thu muc src -> module not found.
- Tac dong:
  - Mat kha nang check version tren tat ca entrypoint CLI.
  - De gay loi tin cay khi publish/CI scripts goi `--version`.

### [Medium] ConfigManager co dau hieu typo/regression o fallback key
- Trieu chung trong code:
  - `['sao.config.json', 'sao.config.json']` (trung lap vo nghia).
  - `paths.saoView || paths.saoView` (khong co fallback key khac).
  - Thong bao loi "sao.config.json or sao.config.json".
- Tac dong:
  - Lam mo logic legacy/new key mapping.
  - Tang rui ro sai cau hinh, kho debug.

### [Medium] Type errors trong output TS example
- Trieu chung trong `examples/js/counter.ts`:
  - Khong resolve duoc module `saola`.
  - Property `__ctrl__` khong co trong type cua class generated.
- Tac dong:
  - Tooling TypeScript canh bao do khi mo file output.
  - Co the anh huong DX neu output TS duoc dua vao build type-check nghiem ngat.

### [Low] Test suite chua bao phu cac regression hien tai
- `npm test` pass nhung bo test khong bat duoc:
  - Loi `--version` cua CLI.
  - Lech schema config giua docs/config local va code validate.
